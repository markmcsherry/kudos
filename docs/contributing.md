# Contributing Guide

Thanks for contributing to Kudos.

## Contribution Flow
- Start with [`index.md`](index.md) for project context and documentation navigation.
- Review requirements in [`ai/specs/requirements.md`](ai/specs/requirements.md).
- Follow product-level UI/UX standards in [`ai/specs/design_spec.md`](ai/specs/design_spec.md).
- For implementation quality, use [`ai/engineering/coding_standards.md`](ai/engineering/coding_standards.md) and [`ai/engineering/common_pitfalls.md`](ai/engineering/common_pitfalls.md).

## AI-Assisted Contributions
- Read [`ai_strategy.md`](ai_strategy.md) first.
- Use [`ai/master_spec.md`](ai/master_spec.md) to assemble task context.
- Include one or more persona docs from [`ai/personas/`](ai/personas/) when you need role-specific review depth.

## Documentation Expectations
- Keep docs focused and concise.
- Update [`ai/decisions/decision_log.md`](ai/decisions/decision_log.md) when making significant technical choices.
- Keep references and paths accurate when moving files.
- Follow [`maintaining_docs.md`](maintaining_docs.md) for docs standards, publishing, and validation.

## Pull Requests
- Clearly describe the problem, approach, and tradeoffs.
- Link related requirements or decision records where applicable.
- Ensure relevant testing is included or explained.
